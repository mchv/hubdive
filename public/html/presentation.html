Code behind the scene

Architecture

The application uses the Play! framework and follow separation of concerns through the MVC pattern.

Model

The data model is made up of 3 classes independent of the hub

Project
A project or repository hosted on a hub. It has notably an owner and an URL.
Committer
A user (registered or not to the hub) who did a Commit on a repository.
Commit
A commit made by a Committer with a message at a date.

Instances of model are retrieved through HubDive. Each hub have a specific implementation of HubDive.


Views

There is one view for each user screen

Index
Display an index page to launch a search.
Search
Display search results and launch a new search.
Show
Display information about a project.

The Index and Search views share the search form through a tag

Controllers

There is one controller for the Application and one for the Api.

Connection

HTTP client

To access hub API, an handmade non asynchronous http client is used

Cache

To cache http response an handmade LRU cache with TTL.
